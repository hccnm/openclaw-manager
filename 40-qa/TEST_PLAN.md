## 1. 测试概述

### 1.1 测试目标

验证 OpenClaw Manager MVP 版本是否满足 PRD 中定义的功能需求和非功能需求，确保系统可以正常启动、运行，并完成基本的实例管理功能。

### 1.2 测试范围

#### 包含范围

| 模块 | 测试内容 |
|------|----------|
| 登录认证 | 用户登录、登出、Token 验证 |
| 服务器管理 | 服务器列表、添加、删除 |
| 实例状态 | 状态查询、刷新状态 |
| 远程操作 | 启动、停止、重启 |
| 前端界面 | 页面渲染、交互、响应式 |
| API 接口 | 所有后端 API 接口 |
| 安全性 | 密码加密、SSH 凭据加密、JWT 认证 |

#### 不包含范围

- 性能压力测试
- 安全渗透测试
- 多用户权限测试（MVP 不支持）
- 批量操作测试（MVP 不支持）
- 实际 SSH 连接测试（需要真实服务器）

### 1.3 验收标准

| 编号 | 标准 | 优先级 |
|------|------|--------|
| AC-001 | 项目可以本地启动运行 | P0 |
| AC-002 | 前后端可以正常通信 | P0 |
| AC-003 | 页面可以展示基础数据 | P0 |
| AC-004 | 具备简单的数据模型和接口 | P0 |
| AC-005 | 用户可以使用 admin/admin123 登录系统 | P0 |
| AC-006 | 用户可以添加/删除服务器 | P0 |
| AC-007 | 用户可以查看服务器列表及状态 | P0 |
| AC-008 | 密码和 SSH 凭据已加密存储 | P0 |

## 3. 测试策略

### 3.1 测试层次

```
┌─────────────────────────────────────────────┐
│                E2E 测试（手动）              │
├─────────────────────────────────────────────┤
│              API 集成测试（手动）            │
├─────────────────────────────────────────────┤
│              单元测试（代码审查）            │
└─────────────────────────────────────────────┘
```

### 3.2 测试类型

| 类型 | 比例 | 说明 |
|------|------|------|
| 功能测试 | 60% | 验证功能是否符合需求 |
| 安全测试 | 20% | 验证加密、认证机制 |
| 兼容性测试 | 10% | 验证多浏览器支持 |
| 代码审查 | 10% | 检查代码结构和质量 |

### 3.3 测试方法

- **黑盒测试**：通过 API 接口和前端界面进行功能验证
- **白盒测试**：代码审查，验证代码结构、加密实现
- **静态分析**：检查文件结构、依赖项

## 5. 缺陷管理

### 5.1 缺陷等级

| 等级 | 描述 | 响应时间 |
|------|------|----------|
| P0 - 致命 | 系统崩溃、数据丢失、安全漏洞 | 立即修复 |
| P1 - 严重 | 主要功能不可用 | 24小时内 |
| P2 - 一般 | 次要功能异常 | 本次迭代内 |
| P3 - 轻微 | UI 问题、优化建议 | 可延后 |
