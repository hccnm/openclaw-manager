## 1. 项目概述

### 1.1 背景

OpenClaw 是一个分布式部署的 AI 助手系统，目前已在多台服务器上运行实例。运维人员需要登录每台服务器手动执行 `openclaw status` 命令来检查实例状态，管理效率低下，缺乏集中监控能力。

### 1.2 目标

开发一个轻量级 Web 管理系统 **OpenClaw Manager**，实现：
- 集中查看所有服务器上的 OpenClaw 实例状态
- 远程执行启动、停止、重启等运维操作
- 提供简单的访问控制机制

### 1.3 技术栈约束

| 层级 | 技术选型 |
|------|----------|
| 前端 | 纯 HTML + CSS + JavaScript（单页面应用） |
| 后端 | Node.js + Express |
| 数据库 | SQLite |
| 通信 | SSH 远程执行命令 |

### 1.4 MVP 范围边界

**包含**：
- 基础登录认证
- 服务器 CRUD（添加/删除/查看）
- 实例状态查询（在线/离线、版本）
- 远程操作（启动/停止/重启）

**不包含**（后续迭代）：
- 多用户权限管理
- 实例日志查看
- 告警通知
- 批量操作
- 操作历史审计

## 2. 目标用户

| 用户角色 | 描述 | 核心诉求 |
|----------|------|----------|
| 运维工程师 | 负责多台服务器上 OpenClaw 实例的日常运维 | 快速查看所有实例状态，远程执行启停操作 |
| 开发人员 | 需要了解 OpenClaw 实例运行状态 | 查看实例是否在线及版本信息 |

**用户画像示例**：
> 小王是一名运维工程师，负责管理 5 台部署了 OpenClaw 的服务器。每天上班第一件事就是逐台 SSH 登录服务器执行 `openclaw status` 检查实例状态。有了 OpenClaw Manager 后，他只需打开一个网页就能看到所有实例状态，需要重启某个实例时一键操作即可。

### 3.2 功能详细说明

#### F001 - 用户登录

**前置条件**：用户未登录

**流程**：
1. 用户访问系统首页
2. 系统显示登录页面
3. 用户输入用户名和密码
4. 系统验证凭据（默认：admin/admin123）
5. 验证通过后跳转至主页，生成会话 Token

**异常情况**：
- 用户名或密码错误：提示"用户名或密码错误"
- 连续错误超过 5 次：锁定 15 分钟（MVP 阶段可选）

**交互要求**：
- 登录表单居中显示
- 支持回车键提交
- 登录按钮有 loading 状态

#### F002 - 登出

**流程**：
1. 用户点击右上角"登出"按钮
2. 系统清除会话 Token
3. 跳转至登录页面

#### F003 - 查看服务器列表

**展示内容**：
| 字段 | 说明 |
|------|------|
| 服务器名称 | 用户自定义名称 |
| IP 地址 | 服务器 IP |
| SSH 端口 | 默认 22 |
| 实例状态 | 在线/离线/未知 |
| OpenClaw 版本 | 如 v1.0.0 |
| 最后更新时间 | 状态最后刷新时间 |
| 操作按钮 | 启动/停止/重启/删除 |

**交互要求**：
- 列表支持按名称/状态排序
- 状态用颜色区分（在线-绿色，离线-红色，未知-灰色）
- 支持响应式布局

#### F004 - 添加服务器

**输入字段**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 服务器名称 | 文本 | 是 | 用于识别，如"生产服务器1" |
| IP 地址 | 文本 | 是 | IPv4 地址 |
| SSH 端口 | 数字 | 否 | 默认 22 |
| SSH 用户名 | 文本 | 是 | SSH 登录用户名 |
| SSH 认证方式 | 选择 | 是 | 密码/私钥 |
| SSH 密码 | 密码 | 条件 | 选择密码认证时必填 |
| SSH 私钥 | 文本域 | 条件 | 选择私钥认证时必填 |

**流程**：
1. 用户点击"添加服务器"按钮
2. 弹出添加表单（模态框或新页面）
3. 用户填写信息并提交
4. 系统验证连接可用性（可选）
5. 保存到数据库，返回列表页

**校验规则**：
- IP 地址格式校验
- SSH 端口范围 1-65535
- 连接测试（可选）

#### F005 - 删除服务器

**流程**：
1. 用户点击服务器行的"删除"按钮
2. 弹出确认对话框："确定删除服务器 [名称] 吗？"
3. 用户确认后，从数据库删除
4. 刷新列表

**安全要求**：
- 必须二次确认
- 删除操作不可恢复（MVP 阶段）

#### F006 - 查看实例状态

**状态定义**：
| 状态 | 说明 | 颜色 |
|------|------|------|
| 在线 | OpenClaw 进程正在运行 | 🟢 绿色 |
| 离线 | OpenClaw 进程未运行 | 🔴 红色 |
| 未知 | 无法连接服务器或命令执行失败 | ⚪ 灰色 |

**状态获取逻辑**：
1. 后端通过 SSH 连接服务器
2. 执行 `openclaw status` 命令
3. 解析命令输出：
   - 包含 "running" → 在线
   - 包含 "stopped" → 离线
   - 连接失败或命令异常 → 未知

**版本获取**：
- 从 `openclaw status` 输出中解析版本信息
- 格式示例：`OpenClaw v1.2.3 (running)`

#### F007 - 刷新状态

**流程**：
1. 用户点击"刷新"按钮
2. 系统重新查询所有服务器状态
3. 更新列表显示

**性能要求**：
- 单服务器状态查询超时：10 秒
- 支持并发查询多台服务器

#### F008/F009/F010 - 启动/停止/重启

**命令映射**：
| 操作 | 执行命令 |
|------|----------|
| 启动 | `openclaw start` |
| 停止 | `openclaw stop` |
| 重启 | `openclaw restart` |

**流程**：
1. 用户点击操作按钮
2. 弹出确认对话框
3. 用户确认后，通过 SSH 执行命令
4. 显示执行结果（成功/失败）
5. 自动刷新状态

**安全要求**：
- 操作前必须确认
- 显示执行结果反馈

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 2 秒 |
| 状态查询响应 | < 10 秒（单服务器） |
| 并发查询 | 支持 5 台服务器同时查询 |
| 会话超时 | 30 分钟无操作自动登出 |

### 4.2 安全需求

| 需求 | 说明 |
|------|------|
| 密码存储 | 使用 bcrypt 哈希存储 |
| 会话管理 | JWT Token，有效期 30 分钟 |
| HTTPS | 生产环境强制 HTTPS（部署要求） |
| SSH 凭据加密 | 数据库中加密存储 SSH 密码/私钥 |
| 操作日志 | 记录所有操作（MVP 可简化） |

### 4.3 可用性需求

| 需求 | 说明 |
|------|------|
| 浏览器兼容 | Chrome 90+, Firefox 88+, Edge 90+ |
| 响应式设计 | 支持 1280px 以上桌面端 |
| 错误提示 | 友好的错误信息，非技术用户可理解 |
| 操作反馈 | 所有操作有明确的成功/失败反馈 |

### 4.4 可维护性需求

| 需求 | 说明 |
|------|------|
| 代码结构 | 前后端分离，目录结构清晰 |
| 配置管理 | 敏感配置通过环境变量管理 |
| 日志记录 | 后端记录关键操作日志 |

## 5. 数据模型

### 5.1 ER 图

```
┌─────────────────┐
│     servers     │
├─────────────────┤
│ id (PK)         │
│ name            │
│ ip              │
│ ssh_port        │
│ ssh_user        │
│ ssh_auth_type   │
│ ssh_password    │ (加密)
│ ssh_private_key │ (加密)

│ version         │
│ last_check_time │
│ created_at      │
│ updated_at      │
└─────────────────┘
```

### 5.2 表结构定义

#### servers 表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 主键 |
| name | TEXT | NOT NULL | 服务器名称 |
| ip | TEXT | NOT NULL, UNIQUE | IP 地址 |
| ssh_port | INTEGER | DEFAULT 22 | SSH 端口 |
| ssh_user | TEXT | NOT NULL | SSH 用户名 |
| ssh_auth_type | TEXT | NOT NULL, CHECK IN ('password', 'key') | 认证方式 |
| ssh_password | TEXT | | 加密存储的密码 |
| ssh_private_key | TEXT | | 加密存储的私钥 |
| status | TEXT | DEFAULT 'unknown' | 实例状态 (online/offline/unknown) |
| version | TEXT | | OpenClaw 版本 |
| last_check_time | DATETIME | | 最后检查时间 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

### 5.3 索引

```sql
CREATE INDEX idx_servers_ip ON servers(ip);
CREATE INDEX idx_servers_status ON servers(status);
```

## 6. 接口设计

### 6.1 API 总览

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /api/auth/login | 用户登录 | 否 |
| POST | /api/auth/logout | 用户登出 | 是 |
| GET | /api/servers | 获取服务器列表 | 是 |
| POST | /api/servers | 添加服务器 | 是 |
| DELETE | /api/servers/:id | 删除服务器 | 是 |
| POST | /api/servers/:id/refresh | 刷新服务器状态 | 是 |
| POST | /api/servers/:id/start | 启动实例 | 是 |
| POST | /api/servers/:id/stop | 停止实例 | 是 |
| POST | /api/servers/:id/restart | 重启实例 | 是 |
| POST | /api/servers/refresh-all | 刷新所有服务器状态 | 是 |

### 6.2 接口详细定义

#### POST /api/auth/login

**请求体**：
```json
{
  "username": "admin",
  "password": "admin123"
}
```

**响应**：
```json
{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "expiresIn": 1800
  }
}
```

**错误响应**：
```json
{
  "success": false,
  "error": {
    "code": "INVALID_CREDENTIALS",
    "message": "用户名或密码错误"
  }
}
```

#### POST /api/auth/logout

**请求头**：
```
Authorization: Bearer <token>
```

**响应**：
```json
{
  "success": true
}
```

#### GET /api/servers

**请求头**：
```
Authorization: Bearer <token>
```

**响应**：
```json
{
  "success": true,
  "data": {
    "servers": [
      {
        "id": 1,
        "name": "生产服务器1",
        "ip": "192.168.1.100",
        "ssh_port": 22,
        "status": "online",
        "version": "v1.2.3",
        "last_check_time": "2026-03-11T10:30:00Z",
        "created_at": "2026-03-10T08:00:00Z"
      }
    ]
  }
}
```

#### POST /api/servers

**请求体**：
```json
{
  "name": "生产服务器1",
  "ip": "192.168.1.100",
  "ssh_port": 22,
  "ssh_user": "root",
  "ssh_auth_type": "password",
  "ssh_password": "your_password"
}
```

**响应**：
```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "生产服务器1",
    "ip": "192.168.1.100",
    "ssh_port": 22,
    "status": "unknown",
    "created_at": "2026-03-11T10:00:00Z"
  }
}
```

#### DELETE /api/servers/:id

**响应**：
```json
{
  "success": true
}
```

#### POST /api/servers/:id/refresh

**响应**：
```json
{
  "success": true,
  "data": {
    "id": 1,
    "status": "online",
    "version": "v1.2.3",
    "last_check_time": "2026-03-11T10:30:00Z"
  }
}
```

#### POST /api/servers/:id/start

**响应**：
```json
{
  "success": true,
  "data": {
    "message": "实例启动成功",
    "status": "online"
  }
}
```

**错误响应**：
```json
{
  "success": false,
  "error": {
    "code": "SSH_CONNECTION_FAILED",
    "message": "无法连接到服务器"
  }
}
```

### 6.3 错误码定义

| 错误码 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| INVALID_CREDENTIALS | 401 | 用户名或密码错误 |
| TOKEN_EXPIRED | 401 | Token 已过期 |
| TOKEN_INVALID | 401 | Token 无效 |
| SERVER_NOT_FOUND | 404 | 服务器不存在 |
| SSH_CONNECTION_FAILED | 500 | SSH 连接失败 |
| SSH_COMMAND_FAILED | 500 | SSH 命令执行失败 |
| VALIDATION_ERROR | 400 | 参数校验失败 |
